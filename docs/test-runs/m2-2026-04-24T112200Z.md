  Phase 0/1 artifacts + `phase-02-surface.md` + `lib/surface-detection.md`

- **Phase 3:** single orchestrator pass (no sub-agent) following
  `phase-03-keystone.md`. Phase 3 is cheap and benefits from keeping the
  Phase 0/2 artifacts in the orchestrator's context.

## Observations for later milestones

1. **Sub-agent correctly found Socket.IO inbound surfaces.** My initial
   orchestrator-pass grep (before spawning the sub-agent) had missed them
   because I scanned `app.ts` and `routes/` but not `lib/startup/`. The
   Phase 2 instructions say to enumerate exhaustively across the partition's
   paths_included; the sub-agent did exactly that and caught 5 WebSocket
   handlers. **Validation:** fan-out at the partition level with an explicit
   "don't sample" instruction produces better coverage than an orchestrator
   pass. Worth reinforcing in the M4 sub-agent prompts.
2. **finale-rest auto-expansion is significant.** Juice Shop wires
   `finale-rest` to 10 Sequelize models and that auto-generates ~70 HTTP
   surfaces. The sub-agent correctly synthesized these (noted in
   `notes: "finale-rest list for X"`). For M4, the auth/authz deep-dive
   needs to understand that `finale.resource(...).all.auth` is the
   centralized check, not scattered per-route middleware. Add this to the
   Phase 5 category 1 grep catalog.
3. **Polyglot assumption was tested minimally.** Juice Shop is a
   TypeScript/Express + Angular codebase; the multi-language dimensions of
   `lib/surface-detection.md` weren't exercised. M3 (scanners) and M4 (deep
   dives) should dogfood against a non-Node target (DVWA for PHP, or a small
   Go MITM repo) to validate the catalog's polyglot coverage.
4. **Handler hashes are populated.** All 171 surfaces have a `handler_hash`
   (non-null). The content-hash normalizer from `lib/handler-hash.md` works
   on Express/Socket.IO code out-of-box. Python and Go hashing will need
   validation on their respective dogfood targets.
5. **Keystone file seeding worked.** Phase 0's `auth.middleware_paths`
   fed the Phase 3 seed list correctly. The grep-expansion added
   `routes/chatbot.ts`, `routes/userProfile.ts`, `routes/verify.ts` — each
   a valid catch.
6. **Config loader heuristic too narrow.** `config/default.yml` made it in
   because the filename matches `config/*.*`. The 13 other profile YAMLs
   (`config/7ms.yml`, `config/addo.yml`, etc.) were **not** flagged — they
   are deployment-specific overrides. Decision to exclude them was correct
   (else the keystone list would be noisy); but the instruction file should
   spell out the "base config only" rule. Minor M2 follow-up — filed as a
   comment in the keystone JSON's `notes[]`.
