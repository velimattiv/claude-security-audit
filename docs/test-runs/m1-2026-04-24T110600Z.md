## Observations for later milestones

1. **Polyglot-aware partitioning is conservative.** Juice Shop has an obvious
   frontend/ (Angular) + backend (Express) split with separate `package.json`
   files but no declared workspaces. The Axis-1 rule (monorepo tool required)
   missed this. Candidate M2 refinement or v2.1: detect "nested-package.json
   without declared workspaces" and treat as an implicit multi-workspace
   partition. **Flagged in `notes[]` on the emitted `partitions.json`.**
2. **Shallow clones break age scoring.** `git clone --depth 1` collapses all
   file introduction dates. The fallback (`age = 1`) is documented but a
   future version should prefer a project-creation-date hint (e.g., the first
   commit on `master` if available) or a config-based override.
3. **LOC heuristic precision.** Without `cloc` or `tokei`, `wc -l` over-counts
   comments and blanks. The profile is correct for rough sizing but should
   not be load-bearing for risk scoring in isolation. Adding a preflight
   suggestion to install one of them would improve accuracy; we stop short of
   requiring them (locked decision 11).
4. **Outbound WebSocket is not an inbound surface.** Juice Shop's ethers
   `WebSocketProvider` usage was correctly classified as an outbound client
   concern (MITM category, Phase 5 #4), not as `surfaces_hint.websocket`. The
   instruction text makes this distinction clearly; worth mirroring the same
   rigor for outbound HTTP clients (fetch, axios, reqwest) in M4.
5. **Existing CodeQL workflow in target.** Juice Shop ships its own CodeQL
   config. Our locked decision is to exclude CodeQL by default; M7 README
   should call out how to *respect* existing user-configured CodeQL rather
   than overriding it.
6. **Graceful degradation worked.** Neither cloc, nor tokei, nor any scanner
   was installed. The workflow's preflight is designed to warn-not-fail; this
   test path validated it end-to-end for M1's narrow scope.
