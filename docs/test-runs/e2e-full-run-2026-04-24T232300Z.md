## What this tells us about the skill

1. **Single-shot `claude -p` orchestration is materially shallower** than a full Phase 5 fan-out would be. The 4 soft misses (alg:none, 2FA, zip-slip, LFI) all need per-route or per-class deep dives that single-shot mode skips. A real production deployment using sub-agent fan-out would catch these.
2. **The artifact contract requires a runtime-side mandate, not just spec language.** Models will read prose and judge that artifacts are optional unless something at runtime says otherwise. `--append-system-prompt` is the lever.
3. **Phase naming drift is consistent across runs.** Orchestrator picks v1-era names (`auth-matrix`, `data-ownership`) regardless of which v2 spec it loaded. That's a model-prior artifact, not a configuration issue.

## v2.1 candidates surfaced

(Already in `docs/ROADMAP.md`; this run confirmed the priority.)

- **External orchestration harness**: drive each phase as a separate sub-agent invocation with explicit artifact validation between, instead of one long `claude -p` call. Forces the contract via shell.
- **Phase-5 fan-out enforcement**: even within `claude -p`, mandate per-category `phase-05-<cat>-<partition>.jsonl` outputs to catch the 4 soft misses.
- **Report template discipline**: orchestrator improvised report shape (no `## Methodology Coverage` etc.); could mandate template adherence in the system prompt.
